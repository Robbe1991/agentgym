2. This is a big update. It has been two years since we launched the first open-source version of AutoGen. We have made 98 releases, 3,776 commits and resolved 2,488 issues. Our project has grown to 50.4k stars on GitHub and a contributor base of 559 amazing people. Notably, we pioneered the multi-agent orchestration paradigm that is now widely adopted in many other agent frameworks. At Microsoft, we have been using AutoGen and Semantic Kernel in many of our research and production systems, and we have added significant improvements to both frameworks. For a long time, we have been asking ourselves: how can we create a unified framework that combines the best of both worlds? Today we are excited to announce that AutoGen and Semantic Kernel are merging into a single, unified framework under the name Microsoft Agent Framework: [https://github.com/microsoft/agent-framework](https://github.com/microsoft/agent-framework "https://github.com/microsoft/agent-framework"). It takes the simple and easy-to-use multi-agent orchestration capabilities of AutoGen, and combines them with the enterprise readiness, extensibility, and rich capabilities of Semantic Kernel. Microsoft Agent Framework is designed to be the go-to framework for building agent-based applications, whether you are a researcher or a developer. For current AutoGen users, you will find that Microsoft Agent Framework's single-agent interface is almost identical to AutoGen's, with added capabilities such as conversation thread management, middleware, and hosted tools. The most significant change is a new workflow API that allows you to define complex, multi-step, multi-agent workflows using a graph-based approach. Orchestration patterns such as sequential, parallel, Magentic and others are built on top of this workflow API. We have created a migration guide to help you transition from AutoGen to Microsoft Agent Framework: [https://aka.ms/autogen-to-af](https://aka.ms/autogen-to-af "https://aka.ms/autogen-to-af").      @everyone This is a big update. It has been two years since we launched the first open-source version of AutoGen. We have made 98 releases, 3,776 commits and resolved 2,488 issues. Our project has grown to 50.4k stars on GitHub and a contributor base of 559 amazing people. Notably, we pioneered the multi-agent orchestration paradigm that is now widely adopted in many other agent frameworks. At Microsoft, we have been using AutoGen and Semantic Kernel in many of our research and production systems, and we have added significant improvements to both frameworks. For a long time, we have been asking ourselves: how can we create a unified framework that combines the best of both worlds? Today we are excited to announce that AutoGen and Semantic Kernel are merging into a single, unified framework under the name Microsoft Agent Framework: [https://github.com/microsoft/agent-framework](https://github.com/microsoft/agent-framework "https://github.com/microsoft/agent-framework"). It takes the simple and easy-to-use multi-agent orchestration capabilities of AutoGen, and combines them with the enterprise readiness, extensibility, and rich capabilities of Semantic Kernel. Microsoft Agent Framework is designed to be the go-to framework for building agent-based applications, whether you are a researcher or a developer. For current AutoGen users, you will find that Microsoft Agent Framework's single-agent interface is almost identical to AutoGen's, with added capabilities such as conversation thread management, middleware, and hosted tools. The most significant change is a new workflow API that allows you to define complex, multi-step, multi-agent workflows using a graph-based approach. Orchestration patterns such as sequential, parallel, Magentic and others are built on top of this workflow API. We have created a migration guide to help you transition from AutoGen to Microsoft Agent Framework: [https://aka.ms/autogen-to-af](https://aka.ms/autogen-to-af "https://aka.ms/autogen-to-af").

     ### victor.dibi&#x61;_&#x20;‚Äî&#x20;_&#x31;0/12/2025 5:39 PMSunday, October 12, 2025 5:39 PM

     LIke @ekzhu mentioned, AutoGen will still be maintained, and you can think of agent framework as a major revision of AutoGen with improvements. Its the same maintainers from AutoGen that are also putting working on Agent Framework and you should see alot of similarities with AutoGen. Below are my top 3 improvement areas that make agent framework a good candidate for your next new project.

     * **Middleware** - write routines that control agent/app behaviour - before/after model calls, function calls, agent runs with the ability to intercept, update data etc. This is critical for security, logging, RAI applications. AutoGen just did not have this.,

     - **Production readiness&#x20;**- this is deeply integrated with Azure Foundry and other services such as openai responses API, Copilot Studio agents etc. You can easily create local agents backed by the Azure AI Agent serve and test them seamlessly. There is also support for deployment for custom workflows on Azure Container apps.,

     -**&#x20;Low level workflow API&#x20;**- many uses want/need an intuitive api that lets them express multi-agent ideas as computational graphs. This is important as it provides control and can improve reliability. The new Workflow API in agent framework makes it quite straightforward to do this. It is similar in spirit to the Graphflow api in autogen but more intuitive (not based on complex broadcast and filtering patterns) For folks who might preview an audio/video overview of the new framework - [https://youtu.be/jyIepE19\_0M?si=1T5napKrOo1SG-I9](https://youtu.be/jyIepE19_0M?si=1T5napKrOo1SG-I9 "https://youtu.be/jyIepE19_0M?si=1T5napKrOo1SG-I9") I also highly recommend the samples folder [https://github.com/microsoft/agent-framework/tree/main/python/samples/getting\_started](https://github.com/microsoft/agent-framework/tree/main/python/samples/getting_started "https://github.com/microsoft/agent-framework/tree/main/python/samples/getting_started") with examples focused on agents, model clients, evaluation, mcp, middleware, observability, threads, tools, workflows.     ### agenticques&#x74;_&#x20;‚Äî&#x20;_&#x37;/17/2025 4:46 PMThursday, July 17, 2025 4:46 PM

    I built a AutoGen-based multi-agent system that mimics the workflow of a data analyst team, with four agents:

    1. Planner (for understanding the business question),
    2. Python Coder (for writing and running analysis),
    3. Executor (for executing the python code in Docker/Jupyter),
    4. Report Generator (for compiling outputs into simple reports),

    As with many AI use-cases, the early results are promising. With a good prompt and relevant data, the system can operate almost on auto-pilot ‚Äî and I‚Äôd say it performs close to what a ~2-year experience data analyst might do. What I liked about AutoGen:

    1. Ability to define agents; and different models by agent In my case: Planner uses Open AI o4-mini; Python coder uses gpt-4.1 and report generator also uses gpt-4.1,
    2. Flexibility of selector function: I wrote a custom selector function for the agent/analysis loop; this was very helpful,
    3. Human in the loop: Data-analysis is very exploratory; so ability to allow human in the loop as part of the interaction was core,
    4. Shared Memory: I was able to define a simple list (shared memory) between the planner and report generator,
    5. Websocket Interaction: I was able to build a custom UI which interacts with AutoGen over websockets,
    6. DockerJupyter Executor: This was one of the core reasons I started exploring AutoGen; being able to execute python code in a isolated docker was very important for this usecase.,

    Overall, I feel AutoGen really helped me to set this up really quickly - without getting in the way. P.S: Launched it here: [https://www.askprisma.ai/](https://www.askprisma.ai/ "https://www.askprisma.ai/")
